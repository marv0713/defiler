# 战国牌策：战斗界面优化文档

版本：v0.1  
适用范围：Web PVE 原型、后续微信小程序与 iOS 移动端适配  
目标：优化战斗界面的信息层级、操作引导、交互反馈，并为后续加入“国策/限定技”与移动端迁移预留空间。

---

## 1. 背景

当前项目是一款类昆特牌/炉石传说的战国题材卡牌策略游戏。第一阶段目标是 Web 端 PVE 对战，后续可能扩展到微信小程序、iOS 海外版本，并逐步加入类 PVP 的异步挑战玩法。

当前战斗界面已经具备基本功能：

- 双方战场区域
- 三排战场：近战、远程、攻城
- 手牌区
- 总分与排分
- 回合/小局信息
- 放弃本轮按钮
- 右侧名词解释与历史记录

但目前界面更像“可运行的调试版”，信息虽然存在，但没有围绕玩家的核心决策组织起来。后续如果加入“国策/限定技”，界面复杂度还会进一步上升，因此需要提前优化结构。

---

## 2. 当前界面主要问题

### 2.1 当前行动提示不够明确

现在界面中间有回合信息和出牌日志，但玩家第一眼不容易判断：

- 现在是不是我的回合？
- 我当前应该做什么？
- 我能不能出牌？
- 我是否应该放弃本小局？
- 这局还差多少分可以赢？

建议将当前行动提示提升为主状态条，而不是隐藏在日志或普通文字中。

推荐文案：

```text
第 1 小局 / 三局两胜
你的回合：请选择一张手牌打出，或放弃本小局
当前比分：你 6 : 6 对手
```

### 2.2 “回合”和“小局”概念容易混淆

当前“第 1 / 3 回合”容易让玩家误解。类昆特牌游戏中至少有三个不同概念：

- 整场对战
- 小局：三局两胜中的第几局
- 行动轮次：双方交替出牌的第几手

建议统一术语：

| 概念 | 推荐命名 |
|---|---|
| 三局两胜中的一局 | 小局 |
| 双方出牌推进过程 | 行动轮 / 出牌轮 |
| 放弃继续出牌 | 放弃本小局 / PASS |

不建议使用“放弃本轮”，因为玩家可能理解成“跳过当前回合”，而不是“本小局不再出牌”。

### 2.3 分数显示分散，战况不够突出

当前双方总分、排分、手牌数、牌库数分布在多个区域。玩家需要来回扫视，才能判断局势。

建议在战场中间增加固定战况条：

```text
第 1 小局｜你的回合｜你 6 : 6 对手｜对手手牌 9｜你手牌 9
```

核心信息优先级应为：

1. 当前比分
2. 当前行动方
3. 当前小局进度
4. 双方手牌数量
5. 是否可 Pass / 是否已 Pass

### 2.4 战场像表格，不够像战场

当前三排区域功能清晰，但视觉上更像数据表。对手区域与我方区域虽然采用镜像排布，但没有足够强的“战线”概念，容易让新手困惑。

建议明确战场结构：

```text
对手后排：攻城
对手中排：远程
对手前排：近战

======== 战线 / Battle Line ========

我方前排：近战
我方中排：远程
我方后排：攻城
```

通过视觉分隔线、背景层级、排名标签强化“双方阵线对峙”的感觉。

### 2.5 右侧信息栏占用空间过大

当前右侧主要显示：

- 名词解释
- 历史记录

问题在于：名词解释并不是战斗中每一秒都需要看的信息，却长期占用大量屏幕空间。真正战斗时更重要的是：

- 当前选中卡牌详情
- 当前可选目标
- 敌方机制/敌人特性
- 最近战斗日志
- 当前国策/限定技状态

建议将右侧区域改为“动态面板”。

未选中卡牌时显示：

```text
敌人机制
最近行动
本小局提示
```

选中卡牌后显示：

```text
卡牌详情
可放置区域
可选目标
预计收益
关键词解释
```

关键词解释不应常驻展开。建议通过点击关键词、悬浮、长按卡牌等方式显示。

### 2.6 手牌操作反馈不足

当前手牌区可以看到卡牌点数与名称，但操作状态不够明显。玩家点击卡牌后，不容易知道：

- 这张牌是否可打？
- 可以打到哪一排？
- 是否需要选择目标？
- 打出后大约产生多少收益？
- 是否存在风险？

建议为手牌增加状态：

| 状态 | 表现 |
|---|---|
| 可打 | 正常高亮 |
| 不可打 | 置灰 |
| 鼠标悬浮 / 触摸按住 | 放大卡牌详情 |
| 已选中 | 边框高亮 |
| 需要目标 | 战场可选目标高亮 |
| 推荐打出 | 轻微提示，不要过度干扰 |
| 高风险操作 | 可显示警告文案 |

点击一张手牌后，可放置的战场排应高亮，其他区域弱化。

### 2.7 放弃按钮语义不清

当前按钮为“放弃本轮”，建议改为：

```text
放弃本小局 / PASS
```

点击后建议二次确认：

```text
确认放弃本小局？
放弃后，你本小局不能再出牌。
```

如果玩家当前领先，可提示：

```text
你当前领先 6 点。放弃后，对手仍可继续追分。
```

如果玩家当前落后，可提示：

```text
你当前落后 4 点。放弃后，本小局大概率失败，但可以保留手牌进入下一小局。
```

这类提示可以帮助新手理解昆特类游戏的核心：赢一小局不一定赚，保留资源也很重要。

### 2.8 历史记录不利于复盘

当前历史记录只显示“谁打出了哪张牌”，但没有显示效果与分数变化。

建议改为：

```text
你：秦国步兵 +6 → 我方总分 6
对手：楚国传令官 +3，召唤楚国术士 +3 → 对手总分 6
```

更完整的日志结构：

```text
行动方：
卡牌名称：
基础点数：
触发效果：
影响区域：
总分变化：
```

### 2.9 缺少敌人机制提示

PVE 战斗中，敌人不应该只是“另一套牌”。每个敌人最好有明确机制，例如：

```text
敌人：楚国传令官
风格：召唤铺场
被动：每打出第 3 张牌，召唤 1 个士卒
反制建议：保留群体伤害或快速抢下前两局
```

敌人机制应该比通用名词解释更优先展示。

### 2.10 后续加入“国策/限定技”会进一步增加复杂度

如果未来加入类似炉石英雄技能 / 昆特领袖技的系统，界面需要展示：

- 阵营 / 主公头像
- 国策名称
- 国策效果
- 是否可用
- 剩余次数
- 是否需要选择目标
- 使用后状态

因此当前页面需要预留技能位置，但不建议第一版就做复杂系统。

推荐第一版只预留：

```text
秦 · 国策：虎狼之师 1/1
楚 · 国策：召令 1/1
```

国策按钮可以放在我方身份区域或手牌区右侧，而不是塞进右侧信息栏。

## 3. 优化目标

本次战斗界面优化目标如下：

1. 玩家一眼看懂当前局势。
2. 玩家清楚知道当前能做什么。
3. 点击卡牌后，明确知道能打到哪里。
4. Pass / 放弃本小局的语义清楚。
5. 右侧信息栏从“百科说明”变成“动态辅助面板”。
6. 为后续加入国策 / 限定技预留位置。
7. 为微信小程序和 iOS 移动端迁移降低成本。
8. 保持 Web 端可快速迭代，不追求过早高保真 UI。

## 6. 移动端迁移挑战

### 6.1 三排战场在竖屏中非常拥挤

当前设计有：

- 对手三排
- 我方三排
- 手牌区
- 分数区
- 右侧说明栏
- 历史记录
- 国策 / 限定技按钮

如果直接迁移到手机竖屏，空间会严重不足。

挑战：

- 卡牌会过小，文字不可读。
- 三排战场高度不足。
- 右侧说明栏无法保留。
- 手牌横向滚动会影响操作。
- 点击目标时容易误触。

### 6.2 右侧面板无法在移动端常驻

移动端不应保留桌面右侧栏。建议改为底部抽屉或弹层：

```text
点击卡牌 → 底部弹出卡牌详情
点击敌人头像 → 弹出敌人机制
点击日志按钮 → 弹出最近行动
点击关键词 → 弹出词条解释
```

### 6.3 手牌操作需要重构

移动端推荐交互：

```text
点击手牌 → 放大预览
再次点击 / 拖拽到高亮区域 → 出牌
长按手牌 → 查看详情
点击空白处 → 取消选择
```

不建议第一版强依赖拖拽，因为小程序和移动 Web 中拖拽手感容易不稳定。

### 6.4 Pass 按钮需要防误触

移动端 Pass 按钮必须远离普通出牌区域，并加确认。可以放在底部右侧，但需要明显区分。

推荐：

```text
[国策 1/1]        [放弃本小局]
```

点击“放弃本小局”后弹窗确认。

### 6.5 横屏与竖屏选择

有两个方向。

#### 方案 A：移动端横屏

优点：

- 更适合三排战场
- 接近当前 Web 布局
- 卡牌和战场空间更充足

缺点：

- 微信小程序中横屏使用门槛更高
- 用户需要旋转手机
- 轻量传播场景可能流失部分用户

#### 方案 B：移动端竖屏

优点：

- 更符合微信用户习惯
- 更容易快速打开
- 更适合轻量 PVE

缺点：

- 三排战场很挤
- 需要压缩信息层级
- 右侧栏必须全部改成弹层
- 手牌和目标选择交互更难

推荐策略：

```text
Web 原型阶段：继续使用横屏布局。
微信小程序阶段：优先评估横屏是否可接受。
如果坚持竖屏，则考虑简化战场排数，例如从三排改为两排。
```

## 7. 国策 / 限定技设计建议

### 7.1 第一版只预留，不要复杂化

当前核心战斗界面还需要优化，不建议马上加入复杂国策系统。

建议阶段：

| 阶段 | 做法 |
|---|---|
| 第一阶段 | 数据结构预留，UI 只预留按钮位置 |
| 第二阶段 | 每个阵营 1 个简单国策，每场限用一次 |
| 第三阶段 | 加入目标选择、条件触发、不同阵营差异 |
| 第四阶段 | 根据数据平衡国策强度 |

### 7.2 推荐国策规则

第一版推荐：

```text
每个阵营一个国策
每场战斗限用一次
无费用
效果简单
部分技能需要选择目标
使用后置灰
```

不推荐第一版做：

```text
每回合可用
冷却系统
能量系统
复杂连携
大量随机效果
```

原因是这会破坏昆特类游戏的资源管理结构，并增加 AI 决策难度。

### 7.3 国策 UI 建议

我方信息区：

```text
🔴 秦国｜国策：虎狼之师 1/1｜手牌 9｜牌库 15
```

手牌区右侧按钮：

```text
[国策：虎狼之师 1/1]
[放弃本小局 / PASS]
```

对手信息区：

```text
🔵 楚国｜国策：召令 未使用｜手牌 9｜牌库 15
```

## 8. 建议实施顺序

### 第一步：修正术语与主状态条

- “回合”改为“小局 / 行动轮”
- “放弃本轮”改为“放弃本小局 / PASS”
- 增加中间主状态条

### 第二步：重构右侧面板

- 默认显示敌人机制 + 最近行动
- 选中卡牌后显示卡牌详情 + 可用目标
- 词条解释改为点击 / 悬浮显示

### 第三步：强化手牌交互

- 增加选中态
- 高亮可放置区域
- 不可用卡置灰
- 需要目标时，高亮目标

### 第四步：优化日志

- 显示卡牌效果与分数变化
- 方便玩家复盘

### 第五步：预留国策位置

- 先放 UI 占位
- 暂不开放复杂技能
- 后续加入每场一次的简单国策

### 第六步：移动端专项设计

- 决定横屏还是竖屏
- 如果竖屏，重新设计卡牌详情、日志、敌人机制的弹层
- 考虑是否将三排简化为两排
